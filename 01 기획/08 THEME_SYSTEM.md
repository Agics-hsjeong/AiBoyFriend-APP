# 🎨 테마 시스템 (Theme System)

> AI 남친 앱의 라이트/다크 테마 정의 및 사용자 테마 설정 규칙. 화면마다 라이트·다크가 뒤섞이지 않도록 **일관된 테마 토큰**으로 통일한다.

---

## 1. 원칙

1. **앱 전체는 하나의 테마(라이트 또는 다크)를 따른다.** 한 화면 안에서 라이트/다크 요소가 섞이지 않는다.
2. 사용자가 **설정에서 테마를 직접 선택**할 수 있다. (시스템 / 라이트 / 다크)
3. **몰입형(시네마틱) 화면**은 분위기를 위해 테마와 무관하게 항상 다크를 유지한다. (영상·음성통화 화면이 항상 어두운 것과 동일한 업계 표준)
4. 포인트 컬러(브랜드 퍼플/핑크)는 두 테마에서 공통으로 유지한다.

---

## 2. 테마 모드 (설정)

`설정 > 화면 및 음성 > 테마`

| 모드 | 동작 |
|------|------|
| **시스템** | OS 다크 모드 설정을 따름 (기본값) |
| **라이트** | 항상 라이트 |
| **다크** | 항상 다크 |

추가로 **테마 색상(포인트 컬러)** 선택: 퍼플(기본) · 핑크 · 블루 · 그린

## 4. 화면별 테마 분류

### A. 테마 전환 대상 (라이트 ↔ 다크 모두 지원)

일상적으로 오래 머무는 **기능형 화면**. 사용자 테마 설정을 따른다.

`홈 · 채팅 · 이상형 생성 위저드(STEP1~7) · 온보딩(02·03·04·06) · 로그인 · 회원가입 · 추억 앨범 · 프로필 · 관계 성장 · 데이트 장소 · 설정`

### B. 항상 다크 (몰입형 / 시네마틱 — 테마 고정)

분위기와 몰입을 위해 테마와 무관하게 **항상 다크**.

`온보딩 01(환영·야경) · AI 생성중 · 첫 만남 · 음성통화 · 3D 모델 뷰어`

> ※ 기획 확정에 따라 B 그룹도 전환 대상으로 옮길 수 있음. 현재는 디자인 시안의 의도(시네마틱)를 반영해 다크 고정.

## 5. 구현 방식 (퍼블리싱/Flutter 공통 가이드)

### 웹 퍼블리싱
- 루트(`<html>` 또는 `.app`)에 `data-theme="light" | "dark"` 속성.
- 모든 색상은 위 **CSS 변수**로만 사용(하드코딩 금지). 테마 전환 시 변수만 교체.
```css
:root, [data-theme="light"] { --bg:#f3f0fb; --surface:#fff; --ink:#2a2440; /* ... */ }
[data-theme="dark"]        { --bg:#140e2b; --surface:#211a40; --ink:#f4f1ff; /* ... */ }
.app { background: var(--bg); color: var(--ink); }
.card { background: var(--surface); border-color: var(--line); }
```
- 몰입형 화면은 `data-theme="dark"` 고정.

### Flutter
- `ThemeData(light)` / `ThemeData(dark)` 정의, `ThemeMode`(system/light/dark)로 전환.
- 색상은 `ColorScheme` + 위 토큰 매핑. `Theme.of(context).colorScheme` 로만 참조.
- 테마 설정값은 `SharedPreferences`(로컬) + 사용자 문서(서버) 저장.

---

## 6. 접근성

- 본문 텍스트 대비비 **4.5:1 이상** (`--ink` on `--bg`).
- 포인트 컬러 위 텍스트는 흰색 고정.
- 다크에서 순흑(`#000`) 대신 딥 네이비퍼플(`#140e2b`) 사용 — 눈부심/번짐 완화.
